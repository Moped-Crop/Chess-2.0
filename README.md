# Chess 2 — ASCENT

Браузерная шахматная игра-вариант. Отличия от классики:

- Доска **10×8** (файлы `a–j`, ряды `1–8`).
- Новая фигура **Петух** — рвётся только вперёд, не отступает.
- Механика **Эволюции**: рабочая фигура (Конь/Слон/Ладья/Петух), завершив ход в дальней зоне соперника, один раз превращается в усиленную форму.
- **MVP:** локальная игра вдвоём за одним устройством (hotseat). Без сервера и ИИ.

В самой игре есть кнопка **«Как играть?»** — интерактивный тур по правилам Петуха и эволюции.

---

## Как запустить

Нужен установленный **Node.js** (версия 18 или новее) — это среда, в которой работают команды ниже. Скачать: <https://nodejs.org> (кнопка LTS).

Открой терминал в папке проекта и выполни по очереди:

```bash
npm install      # один раз — скачать библиотеки (создаст папку node_modules)
npm run dev      # запустить игру
```

После `npm run dev` в терминале появится адрес вида `http://localhost:5173/` — открой его в браузере. Готово, можно играть вдвоём.

Чтобы остановить сервер — нажми в терминале `Ctrl + C`.

## Что реализовано

- **Полные правила Chess 2** (движок): все фигуры, Петух, эволюция в 7 форм, рокировка (Бастион), взятие на проходе, превращение пешки, шах/мат/пат, ничьи (троекратное повторение, правило 75 ходов, недостаток материала).
- **Играбельный интерфейс**: доска 10×8, отрисовка фигур и форм, подсветка ходов и зоны эволюции, выбор формы/превращения, индикаторы шаха и последнего хода.
- **Часы**: пресеты (Пуля/Блиц/Рапид/Классика), инкремент, флаг по времени.
- **Запись партии** (история ходов в нотации) и **автосейв** (партия восстанавливается после перезагрузки).
- **Экспорт/импорт** партии в JSON-файл (бэкап и обмен).
- **Звук** хода/взятия/шаха, **разворот доски**, **цветовые темы** — настройки запоминаются.
- **Обучающий тур** по уникальным механикам.

## Как устроен проект

```
src/
├── engine/   ЧИСТОЕ ЯДРО ПРАВИЛ — не зависит от React/браузера, легко тестируется
│             types, board, attacks, pieces/, moveGen, apply, legality,
│             terminal, hash, special, evolution, setup, index
└── app/      ИНТЕРФЕЙС (React + SVG) — вызывает движок и рисует результат
              store/ (Zustand), components/, clock/, persistence/, tutorial/
tests/        авто-тесты движка и утилит (включая perft — эталонный подсчёт ходов)
```

**Главное правило архитектуры:** `app/` зависит от `engine/`, но `engine/` НИКОГДА не импортирует ничего из `app/`, React или DOM. Это позволяет в будущем подключить ИИ или онлайн без переписывания правил.

Окончательные правила игры зафиксированы в `Rules_Clarification_v1.0.md` (B1–B8) — это источник правды. Технический план — в `Tech_Plan_v1.md`, порядок разработки — в `Pre_Code_Audit.md`. Памятка для ассистента — `CLAUDE.md`.

## Что НЕ входит в MVP (на будущее)

- Онлайн-мультиплеер, ИИ-оппонент, рейтинги.
- Тонкая **балансировка** чисел (номиналы фигур, точные движения форм, граница зоны, порог хода-лимита) — настраивается на плейтесте.
- Расширенный FEN/PGN-экспорт, PWA-оффлайн, второй язык интерфейса.

Архитектура (чистое ядро + слой интерфейса) оставляет дверь для всего этого открытой.
